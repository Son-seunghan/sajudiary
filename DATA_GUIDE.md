# 📊 데이터 저장 위치 & 활용 안내

사주다이어리 사이트에서 어떤 데이터가 어디에 저장되는지, 어떻게 수집·활용할 수 있는지 정리한 문서입니다.

---

## 🔍 현재 상태 (배포 전 / localStorage 기반)

**⚠️ 결론부터: 지금은 데이터를 모을 수 없습니다.**

모든 데이터가 **각 사용자의 브라우저 안**(localStorage)에 저장되기 때문에, 운영자(사용자분)가 다른 사람들의 데이터를 볼 방법이 없습니다.

### 저장 위치별 정리

| 데이터 | 저장 키 | 저장 위치 | 운영자 수집 가능? |
|--------|--------|----------|-----------------|
| 로그인 사용자 정보 | `sajudiary_user` | 사용자 브라우저 | ❌ 불가 |
| 구매 내역 | `sajudiary_purchases` | 사용자 브라우저 | ❌ 불가 |
| 장바구니 | `sajudiary_cart` | 사용자 브라우저 | ❌ 불가 |
| 보유 쿠폰 | `sajudiary_my_coupons` | 사용자 브라우저 | ❌ 불가 |
| 작성한 후기 | `sajudiary_reviews` | 사용자 브라우저 | ❌ 불가 |
| 결제 정보 | (브라우저에 직접 저장 X) | **토스페이먼츠 서버** | 🟡 토스 대시보드에서 확인 |

→ 결제 기록만 **토스 대시보드**에서 보입니다 (토스 가맹점 가입 후).

## 🚀 데이터를 모으려면 — 백엔드(DB) 도입

배포 후 **모든 사용자의 데이터를 한 곳에 모으려면** 백엔드 데이터베이스가 필요합니다. 다음 중 하나를 선택:

### 옵션 1 — Supabase (강력 추천 ⭐)

- **무료 티어**: 500MB DB, 무제한 API 호출, 50,000 MAU
- 한국에서 빠름, 관리 화면 제공
- 회원가입 → 프로젝트 생성 → 테이블 만들기 → API 키 받기 → 코드 연동

```javascript
// 예시: 후기 저장
const { data, error } = await supabase
  .from('reviews')
  .insert({ user_id, product_id, rating, content });
```

**가입**: https://supabase.com

### 옵션 2 — Firebase Firestore

- Google 서비스, 무료 티어 충분
- 한국 속도 보통
- 설정 약간 복잡

**가입**: https://firebase.google.com

### 옵션 3 — Google Sheets API (저비용 간단)

- 후기/주문이 Google Sheets에 행으로 자동 추가
- 운영자는 스프레드시트로 보는 방식
- 구현 간단하지만 트래픽 많으면 한계
- **소규모 시작 시 추천**

## 📈 운영 후 보고 싶은 데이터 (추천)

### 필수 수집 항목

| 데이터 | 활용 |
|--------|------|
| **회원 가입자 수** | 일별/월별 가입 추이, 마케팅 효과 측정 |
| **상품별 구매 내역** | 어떤 상품이 잘 팔리나, 가격 조정 근거 |
| **결제 금액 / 환불** | 실제 매출 |
| **쿠폰 사용률** | 쿠폰이 효과 있는지 |
| **후기 (상품별)** | 만족도, 개선점 |
| **방문 → 구매 전환율** | 마케팅 ROI |

### Google Analytics 연동 (별도 무료)

방문자 행동, 유입 경로, 페이지별 체류 시간 등은 **Google Analytics**(GA4)로 무료 수집 가능합니다. 코드 한 줄 넣으면 자동 시작.

```html
<!-- 모든 페이지 head에 -->
<script async src="https://www.googletagmanager.com/gtag/js?id=G-XXXXX"></script>
```

## 💡 단계별 추천 진행

```
[Stage 1: 베타 - 지금]
  └─ localStorage만 사용 (각자 브라우저에 저장)
  └─ 운영자가 데이터 수집은 못 함
  └─ 직접 베타 테스터에게 후기 받아 카톡으로 회수

[Stage 2: 정식 출시 직후]
  └─ Supabase 도입
  └─ 후기 + 회원 + 구매 → DB로 이전
  └─ Google Analytics 추가

[Stage 3: 트래픽 안정 후]
  └─ 매출/방문/전환 데이터 정기 분석
  └─ 마케팅 자동화 (이메일 발송 등)
  └─ 추후 정기구독, 멤버십 같은 모델 검토
```

## 🔐 개인정보 보호 (중요)

데이터 모을 때 반드시 지켜야 할 한국 법률:

### 개인정보보호법
- **수집 동의** 필수 (가입 시 약관/개인정보처리방침 동의)
- **수집 목적 명시** (예: "후기 게시", "결제 처리")
- **보관 기간 명시** (보통 회원 탈퇴 시 즉시 파기 또는 5년)
- **암호화 의무** (비밀번호, 결제 정보 등)

### 필수 페이지 만들어야 함
1. **이용약관** (서비스 약관)
2. **개인정보처리방침** ← 법적 필수
3. **사업자 정보 표시** (전자상거래법)

→ 배포 직전 단계에서 이 3개 페이지 작성 필수.
→ 표준 양식이 있어 어렵지 않음 (제가 도와드릴 수 있음).
